# Project Brief

## Product Name

Ones n Zeros ERP

## Purpose

Ones n Zeros ERP is intended to become a business operations interface built from an AdminLTE 4 dashboard foundation. The current repository is a static frontend scaffold that should be shaped into a practical ERP application over time.

## Target Users

Primary users are expected to be business operators, administrators, finance staff, inventory staff, sales teams, and managers who need fast access to structured business data.

## Product Principles

- Operational clarity over visual novelty.
- Fast scanning over long-form explanation.
- Predictable navigation over experimental flows.
- Reusable layouts over one-off page designs.
- Clear data states for empty, loading, success, warning, and error views.

## Expected ERP Areas

The final application may include modules such as:

- Dashboard and analytics
- Customers and suppliers
- Sales orders
- Purchase orders
- Inventory
- Invoices and payments
- Projects and tasks
- Users, roles, and permissions
- Reports
- Settings

These modules are not fully implemented yet. Existing AdminLTE demo pages should be converted only when their purpose is clear.

## Current Source Material

The repository includes AdminLTE demo pages for dashboards, forms, tables, widgets, mailbox screens, layouts, profile pages, projects, calendar, kanban, invoice, and settings. These are useful references, but they should not be mistaken for finished ERP features.

## Near-Term Goal

Turn the static template into a clear branded ERP frontend prototype:

1. Establish Ones n Zeros naming and branding.
2. Define the navigation and module map.
3. Convert the main dashboard into an ERP overview.
4. Convert selected pages into real ERP module screens.
5. Remove or archive unused template material after the product direction is stable.
