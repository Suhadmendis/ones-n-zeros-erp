# Roadmap

## Phase 1: Repository Foundation

- Add `.gitignore`.
- Add project Markdown documentation.
- Establish AI agent instructions.
- Identify AdminLTE assets versus product-owned files.
- Keep work on the `dev` branch.

## Phase 2: Branding Pass

- Replace page titles with Ones n Zeros ERP naming.
- Replace sidebar brand text.
- Replace metadata descriptions.
- Replace footer references where appropriate.
- Add project logo assets when available.
- Keep required AdminLTE license notices intact.

## Phase 3: Navigation And Module Map

- Define the ERP sidebar structure.
- Group pages by real business module.
- Remove duplicate demo navigation.
- Keep reference pages until they are no longer useful.

## Phase 4: Dashboard Conversion

- Convert `index.html` into the primary ERP dashboard.
- Replace demo metrics with ERP metrics.
- Define dashboard cards for sales, purchases, inventory, invoices, and tasks.
- Replace sample charts with ERP-relevant chart placeholders.

## Phase 5: Module Screens

Convert selected pages into product screens:

- Customers
- Suppliers
- Items and inventory
- Sales orders
- Purchase orders
- Invoices
- Payments
- Reports
- User management
- Settings

## Phase 6: Technical Hardening

- Decide whether a build system is needed.
- Decide whether repeated layout should become templates or components.
- Add linting or formatting only when the project has a toolchain.
- Add automated checks after the architecture stabilizes.

## Parking Lot

Ideas to revisit later:

- Authentication flow
- Role-based navigation
- API integration
- Theme customization
- Dark mode policy
- Print-friendly invoices
- Offline-ready static demo
